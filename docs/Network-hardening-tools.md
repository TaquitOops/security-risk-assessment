# Network Hardening Tools 

| Security Hardening Task | Description | Common Uses |
|--------------------------|-------------|-------------|
| Baseline Configurations | Documented set of specifications used as a basis for builds, releases, and updates. | Restore system after outage or unauthorized changes. |
| Configuration Checks | Updating encryption standards for stored data. | Detect unauthorized changes to the system. |
| Disabling Unused Ports | Block ports on firewalls, routers, servers to prevent dangerous traffic. | Prevent malicious actors from entering via open ports. |
| Encryption (Latest Standards) | Conceal outgoing data and decrypt incoming data. | Regularly assess encryption strength; update after breaches. |
| Firewall Maintenance | Check and update firewall configurations regularly. | Protect against abnormal traffic and DDoS attacks. |
| Hardware & Software Disposal | Properly wipe and dispose of old hardware/software. | Prevent threats from outdated, unpatched devices. |
| Multifactor Authentication (MFA) | Requires identity verification in two or more ways. | Protect against brute force attacks; set up once, maintain. |
| Network Access Privileges | Permit, limit, or block access to network assets. | Reduce risk of unauthorized access; adjust as needed. |
| Network Log Analysis | Examine logs to identify abnormal traffic/events. | Alert security team; track incidents; SIEM tools. |
| Password Policies | Use hashing/salting per NIST recommendations. | Prevent brute force attacks and password guessing. |
| Patch Updates | OS/software updates to fix vulnerabilities. | Keep systems secure; apply patches promptly. |
| Penetration Test | Simulated attack to identify vulnerabilities. | Protect against potential attacks. |
| Port Filtering | Firewall function to block/allow specific ports. | Control traffic; prevent unauthorized access. |
| Remove/Disable Unused Apps & Services | Eliminate unmaintained applications/services. | Reduce potential vulnerabilities. |
| Server & Data Backups | Store backups locally or in the cloud. | Restore data after attacks, errors, or failures. |
